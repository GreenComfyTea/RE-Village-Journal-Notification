# Links

- **[Nexus Mods](https://nexusmods.com/residentevilvillage/mods/787)**

# Requirements

1. **[REFramework](https://nexusmods.com/residentevilvillage/mods/60)** (**[Nightly](https://github.com/praydog/REFramework-nightly/releases)** 01391+);
2. **[REFramework Direct2D](https://nexusmods.com/residentevilvillage/mods/400)** (v1.0.1+).

# How to install:

1. Install **[REFramework](https://github.com/praydog/REFramework-nightly/releases)**;
2. Install **[REFramework Direct2D](https://nexusmods.com/residentevilvillage/mods/400)**;

> **:pushpin: Windows:** Works normally.

> **:pushpin: Linux:** Latest Experimental Proton seems to have it working normally as well (**https://github.com/cursey/reframework-d2d/issues/5**).

> **:pushpin: Steam Deck:** I've seen people getting it working but I don't have the exact instructions. You will have to figure it out yourself and/or search the nexus pages for information (**[REFramework](https://nexusmods.com/residentevilvillage/mods/60)**, **[REFramework Direct2D](https://nexusmods.com/residentevilvillage/mods/400)**).

3. Download the mod:
   - Official release can be downloaded from **[Nexus Mods](https://www.nexusmods.com/residentevilvillage/mods/787)**;
   - Nightly builds are available in **[this repo](https://github.com/GreenComfyTea/RE-Village-Journal-Notification)** and can contain broken functionality, debugging info on screen, bugs and might require the latest **[nightly build](https://github.com/praydog/REFramework-nightly/releases)** of **[REFramework](https://nexusmods.com/residentevilvillage/mods/60)**. Use with caution!
4. Extract the mod from the archive and copy it into Resident Evil Village folder. Final path should look like this: `/Resident Evil Village BIOHAZARD VILLAGE/reframework/`

> There should be three folders in the reframework folder: `autorun`, `data` and `fonts`.

# How to compile?

**Prerequisites:**

- **[lua-amalg](https://github.com/siffiejoe/lua-amalg)**;
- **[Lua 5.4+](https://lua.org/)**.

**Compilation:**

1. Open `reframework/autorun/compile.bat` and update these variables if your Lua Amalg installation uses different paths:
   - `LUA_EXE` - path to `lua54.exe`;
   - `AMALG_LUA` - path to `amalg.lua`.
2. Run `reframework/autorun/compile.bat`.
3. Optional: run `reframework/autorun/compile.bat --dry-run` to print the generated `lua-amalg` command without compiling.

The script compiles `reframework/autorun/Journal_Notification_dev.lua` into `reframework/autorun/Journal_Notification.lua` and automatically includes all Lua modules found under `reframework/autorun/Journal_Notification/`.
